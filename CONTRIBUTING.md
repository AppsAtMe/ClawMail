# Contributing to ClawMail

Thank you for your interest in contributing to ClawMail!

## Code of Conduct

Please read our [Code of Conduct](CODE_OF_CONDUCT.md). The short version: treat humans and agents alike with the respect and dignity that they deserve.

## How to Contribute

### Reporting Bugs

- Check if the issue already exists
- Provide clear reproduction steps
- Include system information (macOS version, ClawMail version)
- Attach logs if relevant

### Requesting Features

- Open a Discussion first for major features
- Explain the use case
- Consider if it fits ClawMail's philosophy (agent-first, local, private)

### Submitting Changes

1. Fork the repository
2. Create a feature branch
3. Make your changes
4. Add tests if applicable
5. Update documentation
6. Submit a pull request

## Development Setup

```bash
git clone https://github.com/AppsAtMe/ClawMail.git
cd ClawMail
make build
```

See [docs/INSTALL.md](docs/INSTALL.md) for detailed setup.

## Questions?

- Open a [Discussion](https://github.com/AppsAtMe/ClawMail/discussions)
- Check [docs/FAQ.md](docs/FAQ.md)
